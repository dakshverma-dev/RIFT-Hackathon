PROBLEM STATEMENT SUMMARY
--------------------------------------------------------------------------------
Build a web-based Financial Forensics Engine that:
- Accepts CSV file uploads with transaction data
- Detects money muling networks using graph analysis
- Visualizes fraud rings interactively
- Outputs a structured JSON report

Money muling = criminals use networks of individuals ("mules") to transfer
illicit funds through multiple accounts to obscure origin.

--------------------------------------------------------------------------------

--------------------------------------------------------------------------------

- transaction_id (String)

- receiver_id (String)
- amount (Float)
- timestamp (DateTime, format: YYYY-MM-DD HH:MM:SS)

  "fan_in", "fan_out", "shell_network", "high_velocity"

--------------------------------------------------------------------------------
THREE DETECTION ALGORITHMS REQUIRED
--------------------------------------------------------------------------------

1. CYCLE DETECTION (Circular Fund Routing)
   - Detect simple cycles of length 3, 4, and 5 in directed graph
   - Algorithm: DFS-based simple cycle detection
   - Time Complexity: O(V + E)
   - Each cycle = one fraud ring
   - Pattern labels: "cycle_length_3", "cycle_length_4", "cycle_length_5"

2. SMURFING DETECTION (Fan-in / Fan-out)
   - Fan-in: 10+ unique senders → 1 receiver within 72-hour window
   - Fan-out: 1 sender → 10+ unique receivers within 72-hour window
   - Use pandas for timestamp filtering
   - Time Complexity: O(E)
   - Pattern labels: "fan_in", "fan_out"

3. SHELL NETWORK DETECTION (Layered Shell Networks)
   - Chains of 3+ hops
   - Intermediate accounts have only 2-3 total transactions
   - Algorithm: BFS traversal checking node degree
   - Time Complexity: O(V + E)
   - Pattern label: "shell_network"

--------------------------------------------------------------------------------
SUSPICION SCORE FORMULA (0-100, capped at 100)
--------------------------------------------------------------------------------
- Node is in a detected cycle:        +40 points
- Node is fan-in or fan-out:          +30 points
- Node is in shell network:           +20 points
- High velocity (>10 txns in 24hrs):  +10 points

FALSE POSITIVE PROTECTION (CRITICAL):
- If node has >50 total transactions → mark as "legitimate_merchant" → DO NOT flag
- This prevents payroll accounts and merchants from being flagged
